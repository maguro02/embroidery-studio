# 刺繍生成パイプライン仕様（ドラフト）

## 入力前提

- 単色〜十数色のロゴ・線画・アイコンを想定
- 写真からの自動デジタイズは対象外（品質が安定しない）

## ステージ

### 1. 前処理
- リサイズ（実寸換算：mm 指定）
- ノイズ除去（メディアンフィルタ）
- アンチエイリアスの除去（しきい値処理）

### 2. 減色（量子化）
- k-means または median-cut
- 糸色パレット（DMC など）への近似マッピングを将来対応

### 3. 領域分割
- 色ごとにマスク生成 → 輪郭抽出（OpenCV findContours）
- 微小領域の除去（最小ステッチ面積以下）

### 4. ベクター化
- potrace で SVG パス化
- パスの単純化（小さなノイズの平滑化）

### 5. ステッチタイプ割当
| 形状特徴 | 割当 |
|---|---|
| 細長い領域（幅 < 5mm 程度） | Satin |
| 大きな塗り領域 | Fill（タタミ） |
| 1px 線・パス | Run |

### 6. ステッチパス生成
- Fill: 平行線でスキャン、行間 = ステッチ密度（0.4mm 目安）
- Satin: 中心線に沿った振幅、密度・最大幅制限
- 進入点 / 退出点の最適化（移動距離最小化、TSPに近い問題）
- 引き締まり補正（pull compensation）

### 7. 色順序付け
- 同色領域を連続させ、糸替え回数を最小化
- ジャンプステッチ・トリム挿入

### 8. 出力
- pyembroidery で DST/PES/JEF 出力
- プレビュー用に座標列を JSON で別配信

## 既知の難しさ

- 進入退出点の最適化（NP困難系）
- 細部の潰れ（密度が高すぎると糸が浮く）
- 実機での糸切れ・布ヨレ → シミュレーション上は無事故でも実機NGの可能性
